# Panduan Kolaborasi Proyek Unity dengan Teman

Untuk bekerja sama dalam satu proyek Unity dengan teman, Anda memerlukan sistem kolaborasi/kontrol versi (Version Control System). Tanpa sistem ini, jika Anda berdua mengedit proyek secara bersamaan, file proyek akan bentrok dan rusak.

Ada dua metode utama yang paling sering digunakan untuk kolaborasi di Unity:

---

## 🚀 METODE 1: Menggunakan Git & GitHub (Sangat Direkomendasikan)
Metode ini adalah standar industri untuk pemrograman. Gratis, aman, dan memiliki riwayat perubahan yang sangat detail.

### Langkah 1: Buat File `.gitignore` untuk Unity (SANGAT PENTING!)
Unity menghasilkan banyak file sementara yang berukuran sangat besar (seperti folder `Library`, `Temp`, dan `Logs`). Folder-folder ini **tidak boleh** diunggah ke GitHub karena akan membuat proses upload sangat lambat dan merusak proyek teman Anda.

Buat file bernama `.gitignore` di folder utama proyek Anda (sejajar dengan folder `Assets` dan `Packages`), lalu isi dengan baris berikut:

```text
/[Ll]ibrary/
/[T]emp/
/[Obj]/
/[B]uild/
/[B]uilds/
/[L]ogs/
/[U]serSettings/
/[MemoryCaptures]/

# Asset store packages
/[Aa]ssets/AssetStoreTools*

# Autogenerated VS/MD/Consulo solution and project files
ExportedObj/
.consulo/
*.csproj
*.unityproj
*.sln
*.suo
*.tmp
*.user
*.userprefs
*.pidb
*.booproj
*.svd
*.pdb
*.opendb
*.tlog

# Unity virtual file system db
sysinfo.txt

# Builds
*.apk
*.unitypackage
```

### Langkah 2: Menggunakan Git Client (Misal: GitHub Desktop)
Cara termudah bagi pemula adalah menggunakan **GitHub Desktop**:
1. Unduh dan instal [GitHub Desktop](https://desktop.github.com/).
2. Buka GitHub Desktop -> **File** -> **Add Local Repository** -> Pilih folder proyek Unity Anda (`d:\Unity\jedder`).
3. Jika Git belum diinisialisasi, klik **Create a Repository** di folder tersebut.
4. Publish repository tersebut ke GitHub (buat sebagai *Private* jika Anda tidak ingin kode Anda dilihat publik).
5. Buka situs GitHub, masuk ke repository Anda, buka tab **Settings** -> **Collaborators** -> **Add People**, lalu masukkan username GitHub teman Anda.
6. Teman Anda akan menerima undangan, lalu mereka bisa melakukan **Clone** proyek tersebut ke komputer mereka menggunakan GitHub Desktop.

## ☁️ METODE 2: Menggunakan Unity Version Control (Plastic SCM)
Ini adalah fitur resmi dari Unity yang terintegrasi langsung di dalam Editor. Sangat cocok jika teman Anda adalah seorang desainer/artist yang kurang terbiasa dengan Git/GitHub.

### Cara Penggunaan:
1. **Aktifkan di Unity Editor**:
   * Buka proyek Anda di Unity.
   * Di pojok kanan atas Unity Editor, klik tombol **Version Control** (atau buka menu **Window** -> **Version Control**).
   * Klik **Start Using Unity Version Control**.
2. **Undang Teman**:
   * Masuk ke [Unity Dashboard](https://dashboard.unity3d.com/) menggunakan akun Unity Anda.
   * Pilih project Anda -> Masuk ke menu **Users and Permissions** -> **Members** -> Klik **Invite** dan masukkan email akun Unity teman Anda.
3. **Teman Anda Mengunduh Proyek**:
   * Teman Anda cukup membuka **Unity Hub**.
   * Klik tombol **Add** -> **Fetch from cloud** / **Open Remote Project**.
   * Pilih proyek Anda dan tentukan lokasi penyimpanan di komputer mereka.
4. **Cara Kerja**:
   * Di tab **Version Control** di Unity, Anda bisa melakukan **Checkin** (menyimpan perubahan Anda ke cloud) dan **Update** (menarik perubahan terbaru yang dikerjakan teman Anda).

---

## ⚠️ Aturan Emas Kolaborasi Unity (Penting!)
Agar proyek tidak rusak saat dikerjakan bersama:
1. **Gunakan Versi Unity yang SAMA**: Pastikan Anda dan teman Anda menggunakan versi Unity yang persis sama (misalnya `2022.3.x` atau versi yang sedang digunakan saat ini). Perbedaan versi bisa merusak aset (*broken meta files*).
2. **Jangan Mengedit Scene yang Sama Secara Bersamaan**: Jika Anda berdua mengedit Scene yang sama (misal `Scene Level 1`) secara bersamaan, Unity akan mengalami *merge conflict* yang sangat sulit diperbaiki.
   * *Solusi*: Bagi tugas. Satu orang mengedit Scene, satu orang lagi membuat Prefab, atau pisahkan menjadi beberapa Scene berbeda.
3. **Pastikan Selalu Lakukan `Pull/Update` Sebelum Bekerja**: Sebelum mulai mengedit proyek, selalu tarik pembaruan terakhir dari teman Anda agar proyek Anda selalu up-to-date.
